# Support

## Use the right channel

Use a public issue for a reproducible defect in a skill, generator, validator, installer, or
documented compatibility contract. Use Discussions, when enabled, for usage questions,
design exploration, and examples that are not defects.

Use [GitHub private vulnerability reporting](https://github.com/genaptic/skillsets/security/advisories/new)
for credential exposure, malicious behavior, unsafe command execution, release compromise,
or other security impact; see `SECURITY.md`. Use the conduct process for harassment or
community safety concerns.

## Include in a defect report

- Pack and skill.
- Pack version, tag, or commit SHA.
- Client and exact client version.
- Operating system and shell.
- Relevant language, Python, or PostgreSQL version.
- Installation method.
- Minimal prompt or command.
- Expected and actual result.
- Redacted logs or generated output.
- Whether the issue reproduces in a clean directory.

Do not paste credentials, private source, customer data, database dumps, or proprietary
prompts.

## Support boundaries

The project can help with its own content and tooling. It cannot provide production incident
response, database administration, legal or compliance certification, vendor account
support, or guarantees about a model's behavior. Skills are procedures and review aids;
operators remain responsible for approvals, backups, access controls, and changes to real
systems.

No service-level agreement is provided.
